VoiceView.vue — Component Architecture

Two-column layout: vl__sidebar (290px, always visible) | vl__main (flex:1). The sidebar contains voice select, temperature slider, and the instructions panel — always editable when session is idle. The preset dropdown opens upward (bottom-anchored) so it's never clipped.

Noise Floor Calibration
First 30 frames (~1.5s) after capture starts measure ambient RMS. Dynamic threshold = noiseFloor × 2.5 with hysteresis. Exponential smoothing (α=0.6 attack, α=0.15 decay) prevents fast flicker.

voiceLive.js — Three Exports

createCapture(onChunk) — getUserMedia → AudioContext 16kHz → AudioWorklet → returns {stop, stream, ctx}. ctx exposed for mute (suspend/resume without killing the stream). stream exposed for level meter reuse — no second getUserMedia call.

createPlayback() — AudioContext 24kHz. Gapless scheduling via nextPlayTime cursor. flush() drops cursor to ctx.currentTime for barge-in interrupts.

connectLiveWs({...}) — WebSocket binaryType=arraybuffer. Binary frames → onAudio. JSON frames dispatched to onSessionState / onTranscript / onInterrupt / onError.

pcm-capture-worklet.js — Audio Render Thread

Lives in public/ as a plain static file — AudioWorklet.addModule() requires a real URL, not a bundled import. Buffers 128-sample Float32 render quanta until chunkSize (2048 samples = ~128ms at 16kHz), converts to Int16 PCM, posts ArrayBuffer via transferable (zero-copy).

WebSocket Wire Protocol

Single persistent connection per session. Binary frames carry raw Int16 PCM16 audio (browser→server at 16kHz, server→browser at 24kHz). Text frames carry JSON control messages.

Client → Server: {"type":"start","payload":{"model","voice","temperature","instructions"}} then binary PCM then {"type":"end"}

Server → Client: session_state (idle/active) · transcript (role, text, isFinal) · interrupt · error · binary PCM

Session Lifecycle

1. Client sends start frame with config. 2. Server establishes Gemini Live session, responds with session_state{state:"active"}. 3. Browser starts AudioWorklet capture and gapless playback. 4. Bidirectional audio flows until client sends end or WS closes. Gemini's built-in VAD handles all turn detection — no push-to-talk required.

GET /voice/live/config

Returns the list of available voices and the default model ID from the server-side Gemini configuration. VoiceView fetches this on mount; falls back to a hardcoded 29-voice list if the server is unreachable, so the UI is never empty during development.

Go Voice Handler

Upgrades HTTP → WS, reads the start frame, builds LiveConnectConfig{Voice, Temperature, SystemInstruction}, calls genai.Live.Connect(). Then runs two goroutines: (1) read from browser WS → write to Gemini session, (2) read from Gemini → write to browser WS. interrupt message sent when Gemini signals a barge-in.

Gemini Live SDK — google.golang.org/genai v1.57.0

Key surface: client.Live.Connect(ctx, model, *LiveConnectConfig) (*Session, error). Config supports Temperature *float32, Voice string, SystemInstruction *Content. Session is a bidirectional stream — session.Send() for audio input, session.Receive() for output (audio + transcript parts).

Pinia Store — core.js

VoiceView only reads core.baseUrl from the store. All session state (connectionState, messages, audioLevel, etc.) is local to VoiceView — no store pollution. This keeps the voice session isolated and easy to reset.

Design Decisions — UX Rationale
Layout

Two-column sidebar pattern

Config (voice/temp/instructions) stays always visible in the left panel. The right panel is entirely dedicated to the voice experience — orb + state + transcript feed + controls. No tabs, no collapse, no overflow.

UX

Fixed transcript feed

Voice UIs are not chat UIs. The user's attention is on the orb (speaking state), not reading a scrolling log. Three messages in a 120px fixed feed gives "at a glance" context without layout disruption.

Audio

Noise-gated level meter

Standard microphone input has ambient RMS of 0.02–0.06 even in "silence". Without a noise floor, the orb flickers constantly from keyboard/AC/breathing. Calibration + hysteresis makes the orb trustworthy.

Animation

Transform-only orb state

Changing width/height triggers layout reflow which causes the "mirroring/jitter" effect seen previously. Pure scale() transforms are compositor-only — they never touch layout, producing buttery smooth state transitions.

UX

Mute preserves session

toggleMute() calls capture.ctx.suspend()/resume() — this pauses the AudioWorklet without killing the WebSocket. The Gemini session stays alive; the user can unmute and continue speaking without reconnecting.

Resilience

29-voice offline fallback

If /voice/live/config fails (server not running in dev), the full list of 29 known Gemini Live voices populates the selector immediately. The UI is never blank or broken during development.

The noise floor calibration window of 30 frames (≈1.5s) is a good starting point. Consider making this configurable if users report the orb still flickering in noisy environments — a 3× multiplier may be needed for open-office settings.

BE
Backend Lead Backend
The Go handler currently spawns two raw goroutines (read/write) without WaitGroup cleanup. On session end, if the Gemini stream closes before the WS, there's a brief window where sendAudio() can panic on a closed channel. Add context cancellation + WaitGroup.Wait() before returning from the handler.
→ voicelive.go · session goroutines
FE
Frontend Lead Frontend
The bar height formula uses Date.now() inline in the template binding — this causes Vue to re-render the template at 60fps just for the bar animation. Consider moving bar heights to a computed array updated by requestAnimationFrame, or using CSS animation only for the idle pattern and JS only for the active amplitude bars.
→ VoiceView.vue · vl__bars template binding
